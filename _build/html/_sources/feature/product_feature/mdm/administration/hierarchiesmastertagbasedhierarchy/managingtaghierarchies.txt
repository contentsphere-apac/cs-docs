=======================
Manage Tag Hierarchies
=======================
The Manage Tag Hierarchies is an admin feature that sets up tag-based hierarchies wherein each master tag is considered as one level.
This feature allows mixing and matching of tags to build simple hierarchies.


===================     =======================================================================================================================================================================
**Key Term:**           * **Tag:** is a property that is used to describe a product (like *bottle size or color*).

**Description:**        * Hierarchy is concerned with the classification of different objects.
                        * The system allows multiple levels of hierarchies:
                          *Level 1* can be further classified by adding *Level 2* and further classification of *Level 2* can be done by adding more levels under it.

**Related topics:**     * :doc:`Tag Management </feature/product_feature/mdm/administration/datamodeltagmanagement/index>`

**Application:**        * Creating a hierarchy is the same as creating a *Master Taxonomy*.
                          The only difference between the two is that one cannot add a *Property Collection* (collection of attributes and tags) in hierarchies.
                          Adding a *Property Collection* is only done in *Master Taxonomy*.
                        * Hierarchy basically differentiates a *Parent* from a *Child* node.
                        * *Example:*

                            If we create a hierarchy for *Fashion* (Level 1), it can be further classified using a sublevel like *Shirts, Trousers, etc* (Level 2).
                            A *Shirt* can then be further classified based on its *Material or Pattern* (Level 3).

===================     =======================================================================================================================================================================


How to do: Manage Tag Hierarchies
==================================
To manage/create tag hierarchies, follow this set of instructions:

|

#. Log in and go to *Administration*.

.. image:: /feature/_images/mdmimages/cs_managetaghierarchies_login.PNG
|
|


#. Go to the *Data Model* tab and then to *Taxonomies*.

.. image:: /feature/_images/mdmimages/cs_managetaghierarchies_gotothedatamodel.PNG
|
|


#. Under *Taxonomies*, click on **Hierarchies**.

.. image:: /feature/_images/mdmimages/cs_managetaghierarchies_undertaxonomies.PNG
|
|


#. Click on the "+" icon to create a new **Master Hierarchy**.

.. image:: /feature/_images/mdmimages/cs_managetaghierarchies_clickontheicon.PNG
|
|


#. Fill out the required details. Also, slide the button to the right to enable the *Major Taxonomy* option.

.. image:: /feature/_images/mdmimages/cs_managetaghierarchies_fillouttherequireddetails.PNG
|
|


#. Click on the **Create** button.

.. image:: /feature/_images/mdmimages/cs_managetaghierarchies_fillouttherequireddetails2.PNG
|
|


*Note: A Master Hierarchy/Tag has been created.  Further classification can now be done by adding sublevels.*

|
|


#. To add sub-taxonomies, click on the newly created Master Hierarchy and click on the **Add New Level** button found on the next column.

.. image:: /feature/_images/mdmimages/cs_managetaghierarchies_toaddsubtaxonomies.PNG
|
|


#. Type the name of the sublevel and then, click on the **Create 'name'** button.

.. image:: /feature/_images/mdmimages/cs_managetaghierarchies_typethenameofthesubtaxonomy.PNG
|
|


#. Check the details on the pop-up window and click on the **Create** button to continue.

.. image:: /feature/_images/mdmimages/cs_managetaghierarchies_typethenameofthesubtaxonomy2.PNG
|
|


#. Add more sublevels to the hierarchy, as desired.

.. image:: /feature/_images/mdmimages/cs_managetaghierarchies_addmorelevels.PNG
|
|


*Note:  The sublevels have been added and the new hierarchy can be used to tag and describe products.*

|
