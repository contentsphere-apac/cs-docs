=========
Rollback
=========

The Rollback feature allows a user to switch (or to revert) to a previous version of a product in the system.

===================  ================================================================================================================================================================
**Key Term**              * **Rollback**: A feature that enables the user to switch back to a version of a product that he wishes to.
**Description:**          * This is an enhancement to the basic *Rollback* feature, where when the user adds or remove any relationship/context to a product it can be rolled back to the previous version.
                          * This feature allows the user (retailer/supplier) to compare various versions of the product, before deciding which version to rollback to.
                          * This feature allows the user (retailer/supplier) to preview the changes in the product versions.

**Application:**          * The Rollback feature is used to switch back to the desired version of a product.
                          * This is an enhancement to the basic ‘Rollback’ feature, it allows the user add or remove anyproduct relationship/context and return it back to the previous version.
                          * This feature allows the user (retailer/supplier) to compare various versions of the product, before deciding which version to rollback to.
                          *  This feature allows the user (retailer/supplier) to preview the changes in the product versions.
                          * *Example:*

                          Retailer ‘A’ has made several changes in the product namely: deleted an image, changed the language preference, removed an attribute etc. He wants to .The Zoom in feature will help him examine those details across the product versions and compare these versions, before he decides to roll back to the desired one.

How to do: Rollback
=============================
To do this, follow these steps:

#. Go to **PIM**.

.. image:: /feature/_images/mdmimages/cs_rollbackb_clickpim.PNG
|
|

#. Click to select the desired product for which you want to perform rollback operation (in this demo, we select 'Orange bone toy').

.. image:: /feature/_images/mdmimages/cs_rollback_selectproduct.PNG
|
|

#. Click on the *Relationships* tab.

.. image:: /feature/_images/mdmimages/cs_rollbackb_clickrelationship.PNG
|
|

#. In this demo, we will replace the image and roollback to the previous image.

.. image:: /feature/_images/mdmimages/cs_rollbackb_replaceimage.PNG
|
|

.. image:: /feature/_images/mdmimages/cs_rollbackb_replaceimage2.PNG
|
|

.. image:: /feature/_images/mdmimages/cs_rollbackb_replaceimage3.PNG
|
|
#. Click on **Save** TO save to save the changes on the product.

.. image:: /feature/_images/mdmimages/cs_rollbackb_clicksave.PNG
|
|

#. Go to *Relationship* tab.

.. image:: /feature/_images/mdmimages/cs_rollbackb_clickrelationship.PNG
|
|


#.  Go to the *Timeline* tab. The current version of the product would be with the latest version number on the top.

  .. image:: /feature/_images/mdmimages/cs_rollbackb_clicktimeline.PNG
  |
  |

  .. image:: /feature/_images/mdmimages/cs_rollbackb_timeline.PNG
  |
  |

#.  Click on the version before we replaced the image (in this demo, we selected 'version 0 27th March, 2018').

.. image:: /feature/_images/mdmimages/cs_rollbackb_selectversion.PNG
|
|

.. image:: /feature/_images/mdmimages/cs_rollbackb_clickrollback.PNG
|
|

#.    Once the rollback has been confirmed, Relationship Tab opens, and the relationship is restored.View the image (now rereverted to the old version).

.. image:: /feature/_images/mdmimages/cs_rollbackb_viewimage.PNG
|
|
