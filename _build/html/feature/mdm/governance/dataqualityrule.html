DAGOV 6: Data Quality Rule¶

The Data Quality Rule feature provides the functionality to create and set rules in the system to ensure the data quality.

Version: 1.0
User Story:
  • As a user, I want to be alerted in case any data quality violations occur while I am saving an Article so that I can effectively conform to data quality standards for the product.
Description:
  • This feature allows an administrator to create rules to ensure that data quality is maintained in the system.
  • This feature allows the data quality rules to be customized.
  • This feature has two main types for the rules: Rule Violation and Normalization.
    • Rule Violation: alerts a user about a violated rule with an error message.
    • Normalization: reduces data redundancy and improves the data integrity.
Prerequisite:
  • Articles/Products where data quality must be achieved.
Application:

  • Data Quality Rule restricts the entry of unacceptable data in the system.

  • Data Quality Rule reduces the effort in identifying data quality issues in many common information domains.

  • Data Quality Rule customizes the rules for the specific data conditions.

  • Example:

    An administrator does not want to see blacklisted words (or for these words to be added) in the product information pages (e.g. Bomb, Suicide, Shit, etc). In an attribute value (e.g. Description), a rule can be customized to fulfill this condition that will prompt a notification message if not followed.

How to do: Data Quality Rule¶

To do this Data Governance feature, follow these steps:


  1. Go to Settings.


  1. Go to the Rule Configuration section.


  1. Click on the Create button to create a new rule.


  1. Provide a name for the rule and click on the Attributes, Tags, and Roles where the data quality rules will be applied.


  1. Refer to the steps below for a data quality rule applied to an attribute:
  1. Select the attribute (in Cause) and then, add the warning (in Effect). Save all the changes.




  1. Go to Class Configuration.


  1. In the rules section, the list of rules will appear. Select the rule that needs to be applied.


  1. Go to the Home screen.


  1. Go to PIM.


  1. Hover over the ‘+’ icon and select the class.


  1. As indicated in the rule created, the field cannot be left empty. In runtime, if left blank, it will show a warning for the violated rule. When you click on the warning message, the applied rule notification will appear.
