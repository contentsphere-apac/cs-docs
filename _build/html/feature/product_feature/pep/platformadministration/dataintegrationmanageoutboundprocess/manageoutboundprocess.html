Manage Outbound Process¶

Manage Outbound Process is an admin feature that manages Outbound Processes so that data can be evaluated correctly and be transformed to the required output format before exporting.

Description:
  • Outbound Process allows the transfer of system data into an Excel sheet.
  • An outbound process can be used to export multiple articles in one go.
Related topics:
  • Manage Outbound Endpoint
  • Manage System Profile
  • Products
Application:

  • To export data from the system into an Excel sheet, admin uses an outbound process.

  • Example:

    Organization A sells electronics. If a user wants to transfer and export product data into a sheet, the user will make an outbound process with the desired output.

How to do: Manage Outbound Process¶

To do this feature, follow these steps:


  1. Log in and click on Administration.


  1. Click on the Data Integration tab and go to Process.


  1. To create a new process, click on the “+” icon on the right.


  1. Provide the details on the pop-up window and for the Type field, select Onboarding Process.


  1. Click on the Create button.


  1. Click on the newly created process on the list. Set the process using the components based on the requirements.


  1. As an example, drag the Master Article from the Components menu and link it to the Export event. Use other components (Extended Article, Extended DAM, Relationship), as needed.


  1. Click on the Master Article and fill out the details on the right: Sheet Name, Primary Key Column, Type, Class Column.
