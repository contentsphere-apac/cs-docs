Custom ETL Export¶

Custom ETL Export is a feature that allows an admin to manage Custom ETL-based Endpoints so that data can be exported via a custom ETL Process.

Key Term:
  • ETL: Extract-Transform-Load
Description:
  • This feature is used for any modification of a product and also used to save product information on the supplier side.
  • This feature will provide a JSON message of the modified product to any ERP system or any downstream system to update the other parties that use the product. The details will enter the retailer’s system in JSON format and can be viewed in queue section with the destination queue name.
  • The queue will be used by a retailer to read or use the modified information about the product. This information (JSON content) is only visible to the target retailer.
Related topics:
  • Custom ETL Import
  • Manage Outbound Process
  • Manage System Profile
Application:

  • A user can modify data and product information and save these changes.

  • Example:

    Red bull Supplier A wants to update the selling price. The updated price will get to the retailer in JSON format through the JMS queue.

How to do: Custom ETL Export¶

To do this feature, follow these steps:


Note: To export product files via an ETL-based endpoint, the products should be available in the system.


  1. Log in and click on Administration.


  1. Click on the Data Integration tab and go to Workflow.


  1. To create a new process, click on the “+” icon on the right.


  1. Provide the details on the pop-up window and for the Event Type field, select JMS Integration Workflow.


  1. Click on the Create button.


  1. Once clicked, view the window of the newly create process. Set the process using the components based on the requirements.


  1. Add the components such as Export Event, Talend Component, and JMS Producer.



  1. Configure the Talend Component.


    1. Click on the Talend Component.


    1. Configure the component by selecting a Talend job (for this example, it is Export Job) that is available on the right. The job selected will be executed by this component.


  2. Provide the details for the JMS Producer.


    1. Set the following details: Producer IP Address and Producer Port.
      Producer IP Address: this IP address will get through the JMS URL Link. Producer Port: Reserve Port number 61616


    1. Write a Destination Message Queue Name and an Acknowledgement Name that will appear in the JMS queue. With these names, you can receive a message and will also get an acknowlegment that the message is received.


  3. Create endpoint with an Outbound Endpoint type and select the appropriate process.



  1. If all the configurations are ready, go to the selected catalog like PIM, then to Offboarding. Note that permission to access this functionality is set by the Administrator.


  1. Click on All and open the product that is to be updated. Modify the attribute or tag (like List Price, Selling Price, ID, and PID), as needed.


  1. Click on the Save button. The Save button acts as a trigger for the transfer and the export request will start and be processed for the selected product. The dashboard of the Outbound Endpoint will show the modified product.


  1. The product will be transferred in its respective outbound endpoint in the Data Integration catalog. And then, the product’s JSON message will appear in the message queue defined while making the process. To see the message, click on the destination queue name.
