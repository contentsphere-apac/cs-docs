Manage View Templates¶

Manage View Templates lets you set up view templates that are customized to different users.

Key Term:
  • Taxonomy: a taxonomy is a scheme of classification used for an organization’s products.
Description:
  • Easily customize the templates and hide certain portions as necessary.
  • Create templates as per user’s requirements.
  • Show the custom template to the authorized users.
  • Hide certain modules within the product. Even portions of property collections can be hidden from certain users.
Related topics:
  • Manage Permissions
Application:

  • Limit the access of a user to certain information.

  • Hide sensitive information.

  • Example:

    Let’s say the marketing team does not want to show the promotions to a supplier. So what they can do is to create a customized template that does not show promotions and assign it to a role for the supplier to use.

How-to: Manage View Templates¶

Easily manage who gets to see what, and here’s how to do it.


  • Click Administration.


  • Click the Templates button.


  • Click the “+” icon to create a new template. The left pane shows the default templates.


  • Enter the name for the template and click Create.


  • Click the “+” icon and select the desired property collection.




  • Go to the Partner Administration tab, and select a partner from the list. In this demo, we selected “Suppliers”.


  • Click the Roles tab and select a role from the list to which the template is to be assigned.


  • Click the Show Permissions button.


  • Click the Eye icon to make the tab visible or hidden. In the image below, the default image is kept visible and the taxonomy is kept hidden.


  • Select the appropriate class from the list of classes then click the “+” icon to add a template.




  • Select the created template and click Apply.


  • To toggle the visibility of the property, click the Eye icon. To edit the rights for the property, click the Pencil icon.


  • Select the properties to be non-editable (for example, PID) and hidden (for example, Taxonomy).




  • Click Save to save the changes.
