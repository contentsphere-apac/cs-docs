MDM 11: Target and Channel Contextualization¶

Target and Channel Contextualization is a feature that allows users to have attributes that can have multiple values which vary based on context.

Version: 1.0
User Story: As a user, I want to have attributes that can have multiple values which vary based on context.
Description:
  • Target and channel contextualization enables the user to setup values which are based on a context.
  • This feature is particularly useful for values that are dependent on a context or tag.
  • The value can be based on two or more contexts.
Prerequisite:
  • Context configuration
  • Article created
Application:

  • Target contextualization allows the user to have a multi-value attribute based on a tag.

  • Example:

    A product has multiple prices based on the regions it is being sold to, where Region is the context that drives the price attribute.

How to do: Target and Channel Contextualization¶

This How To guide provides instructions on how to have attributes that can have multiple values which vary based on context. To do this, follow these steps:


  1. Go to Settings tab.


  1. Select Context Configuration.


  1. Create new context.


  1. Fill in the name and select Attribute Context as the feature type.


  1. Select tag group to be added in the context and click Save.


  1. In the class configuration, select the class.


  1. Select the desired attribute then select the context from the drop-down list.


  1. Click Save and go to Runtime.


  1. Select the product.


  1. Click on the attirbute with variations.


  1. Create Attribute Variant for list price.


  1. Select the tag group, enter the value for the attribute, and click Save.
