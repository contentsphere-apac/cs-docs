Process Flow Check¶

Having a process flow feature provides the user a visual representation of the entire onboarding process. Data violations, if any, are depicted in red/orange color. A successful data onboarding process is shown in green color.

Prerequisites¶

  • Configured Partners
  • Configured Users

Key Terms¶

  • Workflow - A Workflow defines the structure and components of a file that is to be imported into the system or exported from the system. It provides all the details of all the components configured in a sheet. The system will interpret the imported file in a way we define the details in the Process section.

How to do a Process Flow Check¶

  1. Log in to the application.

  1. Click on the Administration icon.

  1. Navigate to Data Integration -> Workflow menu.

  1. Click on the “+” icon to create the workflow.


  1. Enter the details such as Name, Event Type, and Code, then click on the Create button.

  1. Select the created workflow. On the pop-up window that appears, configure all the components of the workflow.


  1. The Graph section in the pop-up window is divided into three vertical sections. The first two vertical sections are called Workflow Modeler.
It allows to configure workflows using BPMN 2.0 modelling notation. This will allow users to model complex onboarding process using universally known and standardized BPMN 2.0 notation. The third vertical section contains the details of each component of onboarding file.

  1. Drag and drop the notations in Workflow Modeler to define onboarding process. The workflow starts with the Start Event. It acts as a process trigger. So, first element in the workflow will be the Start Event. .

  1. Select the Start Event notation and all the options that you can add after it, will appear on the screen.

  1. Start adding the notations to define a workflow according to onboarding file. For example, if you want to add Article in a flow, then click on Task notation. Custom actions for different users will be managed through workflow Task.

  1. Once you click on the icon, the task gets added in the flow.

  1. Adding Task notation simply defines the task. You need to define what kind of a task it is, whether it is import article, import asset, import relationship, etc. You can do this by clicking on the icon that looks like plier.

  1. As soon as you click on Wrench icon, a pop window appears containing types of task.

  1. If you are defining article, then select Article Import option from the list.

  1. Double-click on the created task icon and enter the name for that icon.

  1. Define the components of the system such as Assets, Relationships, Nature Classes, Create a task, click on the Wrench icon, and give it a name.

  1. Once you add all the components in a flow, append the End Event notation at the end. It represents the end of flow. Select the last component and click on the End Event notation to append the End Event at the end.

  1. View the end event icon appended at the end of flow.

  1. Click on each component and fill in the details of each component in third vertical section like Sheet Name, Primary Key Column, Type, Class ID/Class Column, Multiclassification, Type, Select Classes/ Class Column Name, Taxonomies and Click on SAVE button.
