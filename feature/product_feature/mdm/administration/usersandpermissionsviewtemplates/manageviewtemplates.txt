======================
Manage View Templates
======================
Manage View Templates lets you set up view templates that are customized to different users.


===================     =============================================================================================================================================================
**Key Term:**           * **Taxonomy:** a taxonomy is a scheme of classification used for an organizationâ€™s products.

**Description:**        * Easily customize the templates and hide certain portions as necessary.
                        * Create templates as per user's requirements.
                        * Show the custom template to the authorized users.
                        * Hide certain modules within the product. Even portions of property collections can be hidden from certain users.

**Related topics:**     * :doc:`Manage Permissions </feature/product_feature/mdm/administration/inheritanceanddatatransfer/linkingobjects>`

**Application:**        * Limit the access of a user to certain information.
                        * Hide sensitive information.
                        * *Example:*

                            Let's say the marketing team does not want to show the promotions to a supplier.
                            So what they can do is to create a customized template that does not show promotions and assign it to a role for the supplier to use.
===================     =============================================================================================================================================================


How to do: Manage View Templates
=================================
Easily manage who gets to see what, and here's how to do it.

|

#. Click the **Gear** icon to go to **Settings**.

.. image:: /feature/_images/mdmimages/viewtemplates_settings.PNG
|
|


#. Click the **Template Configuration** button.

.. image:: /feature/_images/mdmimages/viewtemplates_templateconfig.PNG
|
|


#. Click the "+" icon to create a new template. The left pane shows the default templates.

.. image:: /feature/_images/mdmimages/viewtemplates_create.PNG
|
|


#. Enter the name for the template.

.. image:: /feature/_images/mdmimages/viewtemplates_entername.PNG
|
|


#. Click the **Eye** icon to make the tab visible or hidden. In the image below, the default image is kept visible and the taxonomy is kept hidden.

.. image:: /feature/_images/mdmimages/viewtemplates_eyeicon.PNG
|
|


#. Click the "+" icon to display a list of all property collections.

.. image:: /feature/_images/mdmimages/viewtemplates_viewpropcoll.PNG
|
|


#. Select the appropriate property collections and click **Save**.

.. image:: /feature/_images/mdmimages/viewtemplates_save.PNG
|
|


#. Click the **Permissions Configuration** button and select a role to which the template is to be assigned.

.. image:: /feature/_images/mdmimages/viewtemplates_permission.PNG
|
|


#. Select the appropriate class from the list of classes then click the "+" icon to add a template. A green check mark indicates default template.

.. image:: /feature/_images/mdmimages/viewtemplates_selectclass.PNG
|
|


#. Add the template in the class and click **Apply**. On the top right, click **Templates**.

.. image:: /feature/_images/mdmimages/viewtemplates_clickapply.PNG
|
|


#. Select the template.

.. image:: /feature/_images/mdmimages/viewtemplates_selectclass.PNG
|
|


#. To toggle the visibility of the property, click the **Eye** icon. To edit the rights for the property, click the **Pencil** icon.

.. image:: /feature/_images/mdmimages/viewtemplates_edit.PNG
|
|


#. Select the properties to be non-editable (for example, *PID*) and hidden (for example, *Taxonomy*).

|

#. Set the template as default for the class and click **Save**.

.. image:: /feature/_images/mdmimages/viewtemplates_setasdefault.PNG
|
|


#. The image below shows the template where the Taxonomy is hidden and the Pricing section is not editable.

.. image:: /feature/_images/mdmimages/viewtemplates_final.PNG
|
|
